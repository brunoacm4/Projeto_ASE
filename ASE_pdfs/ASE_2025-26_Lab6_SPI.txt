2. Introduction
The Serial Peripheral Interface (SPI) bus is essential in the embedded systems
ecosystem, enabling fast and efficient communication between a microcontroller
and a wide variety of peripherals – from sensors and actuators to communication
modules and storage devices. Its structural simplicity, based on a few signal lines,
makes it particularly suitable for integrating multiple components into a single
system, maintaining low cost and high reliability.
On platforms like the ESP32-C6, SPI is frequently used to connect measurement
sensors, interface controllers, expansion modules, and memory cards. This versatility
makes it a strategic tool for developing IoT and automation applications, where it is
common to combine data acquisition, local processing, and storage.
The use of SD cards via SPI adds an important layer of functionality: persistent local
storage. This allows, for example, saving sensor logs, operating events, or system
configurations, even in environments without connectivity. Furthermore, SPI access
simplifies integrating recording and storage solutions into industrial research or

SPI, including read and write operations on SD cards, is therefore a fundamental step
in building robust, scalable, and autonomous embedded systems that operate
efficiently and reliably in real-world scenarios.
In this class, we will explore using the SPI protocol to read and write data to an SD
card.

Figure 1 - Adafruit Display (TFT View – left; SD card reader view - right).
2

As can be seen, this display has a set of pins that allows it to operate simultaneously
as a display and as an SD card reader. The available pins for connection and their
functions can be summarized as follows:
•

Vin (+3V to +5V) – power supply pin; must be connected to +3V to +5 V DC.

•

3V – +3V (approx.) output from the display integrated voltage regulator
(leave unconnected).

•

GND – ground pin, common to power supply and signals.

•

SCK – SPI clock input pin (accepts voltage levels between +3V and +5 V).

•

MISO – SPI Master Input (microcontroller in) Slave Output (module out) pin,
used by the SD card, not used by the TFT display - write only (3.3 V logic
output).

•

MOSI – SPI Master Output (microcontroller out) Slave Input (module in) pin,
used to send data from the microcontroller to the SD card or TFT display
(accepts voltage levels between +3V and +5 V).

•

TFTCS – SPI chip select pin of the TFT display, used to write into the display
(accepts voltage levels between +3V and +5 V).

•

RST – TFT display reset pin, connect to ground to reset the display, includes
an automatic reset circuit when power is turned on (accepts voltage levels
between +3V and +5 V).

•

DC (Data/Command) – TFT display data or command selector pin (accepts
voltage levels between +3V and +5 V).

•

SDCS (SD Card CS) – SPI chip select pin of the SD card, used to read/write
data from/to the card (accepts voltage levels between +3V and +5 V).

•

LIT – PWM input pin for backlight control of the display, by default it is at a
high level (backlight on), it can be controlled by PWM signal or set to a low
level to turn off the backlight (accepts voltage levels between +3V and +5 V).

In this lab, we will only use the SD card reader feature. For this reason, the TFTCS pin
must be connected to 3.3V to keep the display inactive during the various exercises.

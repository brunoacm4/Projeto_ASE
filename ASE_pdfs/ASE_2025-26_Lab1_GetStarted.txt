2.1 ESP32-C6-DevKitC-1 Board
The ESP32-C6-DevKitC-1 is based on the ESP32-C6-WROOM-1(U) module, a generalpurpose module with 8 MB of SPI flash. Most of the I/O pins are available on the
board's bottom headers, facilitating peripheral connections. Development with this
board can be done by mounting the ESP32-C6-DevKitC-1 on a breadboard and
connecting peripherals to it. A view of the board is shown in Figure 1.

Figure 1 - ESP32-C6-DevKitC-1 view.
Figure 2 depicts the structure of the ESP32-C6-DevKitC-1. It integrates Wi-Fi 6
functionality in the 2.4 GHz band, Bluetooth 5, and IEEE 802.15.4 (Zigbee 3.0 and
Thread 1.3). All available General-Purpose Input/Output (GPIO) pins (except the SPI
bus for the flash) are provided on the board headers. The board also features a 5V to
3.3V low-dropout (LDO) voltage regulator, a power-on LED, and a USB<->UART
converter that supports transfer rates up to 3 Mbps.
The ESP32-C6-DevKitC-1 has a USB Type-C port compatible with USB 2.0, capable of
transferring up to 12 Mbps. This port powers the board, writes firmware,
communicates with the module using USB protocols, and supports Joint Test Action
Group (JTAG) debugging. The board also includes a download (Boot) button, a reset
button, and an addressable RGB LED driven by GPIO8. There is even a J5 connector

interfaces such as UART, I2C and SPI, as well as analog pins for reading sensors.
There are also dedicated pins for special functions, such as Pulse Width Modulation
(PWM) for motor and servo control, as well as general-purpose inputs and outputs

3. ESP-IDF Platform (Compilation Toolchain)
The ESP-IDF (Espressif IoT Development Framework) is a popular platform for
developing IoT solutions on the ESP32-C6-DevKitC-1 module, providing a
comprehensive development environment aligned with the module's capabilities.
ESP-IDF is designed to leverage the ESP32-C6's features, offering a complete set of
drivers, libraries, and examples. Its integration allows development to focus on the
core of embedded and IoT applications, avoiding the low-level complexities of
configuring and controlling peripherals. Additionally, this platform is open-source
and reasonably well-documented, with an active community of users and developers
that facilitates the discovery of solutions, troubleshooting, and the adoption of
proven practices throughout the development cycle. Another significant advantage
of ESP-IDF is its native support for advanced connectivity features, including Wi-Fi 6,
Bluetooth 5, and low-power network protocols such as Zigbee and Thread. This
integrated approach greatly simplifies deploying IoT applications that require access
to multiple communication technologies.

3.1 Installation and Configuration
The first step in your journey with the ESP32-C6-DevKitC-1 module is to install ESPIDF, which will allow you to edit the IoT application firmware, compile the source
code, program the module, and perform debugging and testing. To do so, you must
strictly follow the instructions provided on the Get Started page for the ESP32-C6
module from Expressif, where you will find a step-by-step guide on how to install the
platform:

https://docs.espressif.com/projects/esp-idf/en/v5.5.2/esp32c6/get-

started/index.html. Independently, if you are installing on MS Windows, Linux, or
macOS, you must install it for command-line (terminal) use as well as for use with
the VS Code Extension (after installing the base ESP-IDF, also install the VS Code
extension with the same name).

4. Testing of ESP-IDF Get Started Projects
The following subsections describe the major steps for compiling and running two
example projects provided with ESP-IDF to test two board functionalities: terminal
output and digital outputs. Both must be compiled and tested using both the
command line and the VS Code Integrated Development Environment with the ESPIDF extension installed.

4.1 Hello World Application
After installing the development environment, the next step is to create a basic
application to test the entire ecosystem.
Create a copy in your home folder of the “Hello World” example located in:
”C:\Espressif\frameworks\esp-idf-v5.5.2\examples\get-started\hello_world”
(MS Windows path)
Then compile, flash and execute the example in the ESP32-C6 board. Finally,
carefully analyse the source code against the application's output.

6

The following commands must be executed in the terminal, in the project root folder
(after copying the example):
idf.py set-target esp32c6 (to select a particular device family)
idf.py build (to compile and link the application)
idf.py -p <SERIAL PORT> flash (to write the application image into the
kit FLASH memory)
idf.py -p <SERIAL PORT> monitor (to open the terminal to visualize the
application output)
The following command can also be used to configure the project settings:
idf.py menuconfig (if the project settings are changed, a complete build must
be performed)
Repeat the procedure of “Hello World” project creation, compilation, and test inside
the VS Code using the ESP-IDF extension (which must be installed prior to project
creation).

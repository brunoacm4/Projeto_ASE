2. Introduction
In the context of the Internet of Things, it is increasingly important that devices allow
for clear and efficient interaction with users. Human-Machine Interfaces (HMIs) are
fundamental to making the information generated by intelligent systems more
accessible and understandable.
This guide aims to demonstrate this interaction using the ESP32-C6 with an Adafruit
0.96-inch TFT screen. Through this combination, the intention is to create a simple,
functional visual interface that presents real-time information, improves interaction
between the user and the device, and enhances access to the device's context.

3. Practical Work
The practical work in this lesson is based on the driver and graphics library available in
the ST7735-ESP32-C6 repository. Additionally, the technical data sheet for the
Adafruit 160x80 0.96" display, available on the course e-learning page, should be
consulted.
The ST7735-ESP32-C6 repository provides a modular, efficient foundation for
communicating with the TFT screen via the SPI interface, including functions for
initialization, color control, drawing basic shapes, text display, and updating screen
content. The code structure also allows for easy extension, facilitating the creation of
interactive menus, real-time data graphs, or sensor status indicators, all of which are
typical elements of a functional HMI.

Question 6: Using a push button, create a graphical interface that toggles between
two screens. The default screen is the one developed in the previous question. The
alternative screen should show the latest system log messages, one per line (up to
five lines in total). Whenever the user switches between screens by pressing the
button, "next" should be displayed. Note: there should be support for a list of more
than 5 messages, observing the simulated evolution of the logs over time.
Question 7: Following question 6, create an additional screen where the temperature
graph over time can be viewed in real-time. The axes of the graph should be marked.
Points should be presented starting from the origin, that is, t=0 represents the instant
when the user “entered” the screen, with the temperature values obtained from then
on represented in the graph. The functionality of rotating between screens using the
push button should remain operational.

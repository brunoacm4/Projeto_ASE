2. Introduction
In the Internet of Things (IoT) and embedded systems, the ability to control and read
electrical signals via digital and analog input/output ports is fundamental to creating
intelligent, interactive devices. Knowing how to program these ports allows a
microcontroller to communicate with sensors, actuators, and other peripherals,
transforming signals from the physical world into digital data that can be processed
and transmitted, or converting digital commands into concrete actions, such as
turning on a light, moving a motor, or regulating the intensity of an LED. This ability
not only provides direct control over the environment but also enables automation,
real-time monitoring, and the integration of multiple devices into an IoT network.
Understanding the operation and programming of I/O ports is therefore essential for
developing robust, efficient, and responsive solutions, allowing creators to transform
ideas into functional applications that intelligently interact with the world around
them.

3.1 Controlling an LED
The most basic step in any embedded system is to program a digital output to
alternate between the "1" and "0" states with a given period. Next, the "skeleton" of
a functional program is provided, which, combined with the circuit in Figure 1a),
allows the respective LED to be activated periodically.
#include <stdio.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "driver/gpio.h"
#define BLINK_GPIO

GPIO_NUM_15

// GPIO15 on ESP32-C6

void app_main(void)
{
gpio_config_t ioCfg = { // configure GPIO15 as output
.pin_bit_mask = 1ULL << BLINK_GPIO, // bit mask for GPIO15
.mode = GPIO_MODE_OUTPUT,
.pull_up_en = GPIO_PULLUP_DISABLE,
.pull_down_en = GPIO_PULLDOWN_DISABLE,
.intr_type = GPIO_INTR_DISABLE
};
gpio_config(&ioCfg);
bool level = false;
while (1)
{
gpio_set_level(BLINK_GPIO, level);
level = !level; // invert for next toggle
vTaskDelay(pdMS_TO_TICKS(1000)); // delay 1 second
}
}

3.3 Reading the state of a push button (interrupt)
When using interrupts in embedded systems, the challenge of securely transferring
information between the interrupt service routine (ISR) and the main application
always arises. Implementing operations such as printing within an ISR is not
recommended, as it can delay other important events. Queues offer an efficient and
secure mechanism for this communication. The ISR can quickly queue a small piece
of data, such as an event signal or a sensor reading, and return it immediately. Later,
a FreeRTOS task, running outside the interrupt context, retrieves this information
from the queue and processes it without time constraints. In this way, ISRs remain
lightweight while ensuring reliable and synchronized data transfer between interrupt
events and application logic.
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "freertos/queue.h"
#include "driver/gpio.h"
#define INPUT_GPIO

GPIO_NUM_23

// input pin example

static QueueHandle_t gpioEvtQueue = NULL;
// ISR handler
static void IRAM_ATTR GPIOISRHandler(void* arg)
{
uint32_t gpioNum = (uint32_t)arg;
// Send GPIO number to the queue (from ISR context)
xQueueSendFromISR(gpioEvtQueue, &gpioNum, NULL);
}

3.4 Controlling a PWM Output
The ESP-IDF LEDC library is the tool of choice for generating precise, efficient PWM
signals on the ESP32 family of microcontrollers. Through it, it's possible to control
the intensity of an LED by adjusting the signal's duty cycle, allowing, for example, the
creation of smooth fade effects or simply regulating the brightness at different
levels. In the case of an LED, LEDC enables a simple digital pin to act as a dimmer,
opening the way for more dynamic and interactive interfaces in IoT projects and
embedded systems.
#include <stdio.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "driver/ledc.h"
#define LED_GPIO

GPIO_NUM_15

#define LEDC_CHANNEL

LEDC_CHANNEL_0

#define LEDC_TIMER

LEDC_TIMER_0

#define LEDC_MODE

LEDC_LOW_SPEED_MODE

#define LEDC_DUTY_RES

LEDC_TIMER_13_BIT

#define LEDC_FREQUENCY 5000

// 13-bit resolution
// 5 kHz PWM

8

void app_main(void)
{
// 1 - Configure timer
ledc_timer_config_t ledcTimer =
{
.speed_mode = LEDC_MODE,
.timer_num = LEDC_TIMER,
.duty_resolution = LEDC_DUTY_RES,
.freq_hz = LEDC_FREQUENCY,
.clk_cfg = LEDC_AUTO_CLK
};
ledc_timer_config(&ledcTimer);
// 2 - Configure channel
ledc_channel_config_t ledcChannel =
{
.gpio_num = LED_GPIO,
.speed_mode= LEDC_MODE,
.channel = LEDC_CHANNEL,
.intr_type = LEDC_INTR_DISABLE,
.timer_sel = LEDC_TIMER,
.duty = 0,
.hpoint = 0
};
ledc_channel_config(&ledcChannel);
uint32_t maxDuty = (1 << LEDC_DUTY_RES) - 1;
uint32_t duty10 = maxDuty / 10;
while (1)
{
ledc_set_duty(LEDC_MODE, LEDC_CHANNEL, duty10);
ledc_update_duty(LEDC_MODE, LEDC_CHANNEL);
vTaskDelay(pdMS_TO_TICKS(1000));
}
}

3.5 ADC Configuration and Use in One-Shot Mode
The ADC's one-shot operating mode is used when precise, point-in-time readings of
analog signals are required, without continuously maintaining active conversion. This
mode allows the processor to request a measurement only when necessary,
reducing power consumption and freeing up system resources, which is especially
important in battery-powered IoT devices. In addition, one-shot simplifies
integration with specific tasks, such as monitoring sensors at defined intervals or
reading critical values in response to events, ensuring efficient, accurate processing
of analog signals within embedded architectures.

#include "freertos/FreeRTOS.h"
#include "freertos/task.h"

ADC_UNIT_1

.unit_id = ADC_UNIT

adc_oneshot_new_unit(&adc1InitCfg, &adc1Handle);
